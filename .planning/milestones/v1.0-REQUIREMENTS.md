# Requirements: Objetiva Speecher v1.0

**Defined:** 2026-02-06
**Core Value:** Instant, reliable voice-to-cursor flow under 2 seconds with zero manual intervention

## v1 Requirements

Requirements for initial release. All shipped.

### Backend Server

- [x] **BACK-01**: Backend accepts HTTP POST /transcription with deviceId and text
- [x] **BACK-02**: Backend maintains WebSocket server for agent connections
- [x] **BACK-03**: Backend maintains connection registry (deviceId -> WebSocket mapping)
- [x] **BACK-04**: Backend routes transcriptions to correct agent via WebSocket
- [x] **BACK-05**: Backend provides GET /devices API listing connected agents
- [x] **BACK-06**: Backend handles agent registration on WebSocket connect
- [x] **BACK-07**: Backend removes agents from registry on disconnect
- [x] **BACK-08**: Backend implements heartbeat ping/pong (20-30s intervals)

### Voice Input & Recognition

- [x] **VOICE-01**: Mobile app provides tap-to-start voice recording button
- [x] **VOICE-02**: Mobile app provides tap-to-stop voice recording function
- [x] **VOICE-03**: Mobile app integrates Android SpeechRecognizer with es-AR locale
- [x] **VOICE-04**: Mobile app displays partial results during recognition
- [x] **VOICE-05**: Mobile app displays final transcription result
- [x] **VOICE-06**: Mobile app shows connection status indicator (connected/disconnected)
- [x] **VOICE-07**: Mobile app handles all SpeechRecognizer error codes gracefully
- [x] **VOICE-08**: Mobile app auto-reconnects to backend when connection drops

### Device Management

- [x] **DEV-01**: Mobile app discovers backend server via mDNS
- [x] **DEV-02**: Mobile app fetches list of connected devices from backend
- [x] **DEV-03**: Mobile app displays devices by hostname in selection list
- [x] **DEV-04**: Mobile app allows user to select target device
- [x] **DEV-05**: Mobile app persists selected device across app restarts
- [x] **DEV-06**: Mobile app updates device list when devices connect/disconnect

### Desktop Agent (Windows)

- [x] **WIN-01**: Windows agent connects to backend via WebSocket
- [x] **WIN-02**: Windows agent registers with hostname-based deviceId
- [x] **WIN-03**: Windows agent receives text via WebSocket messages
- [x] **WIN-04**: Windows agent writes text to clipboard using clipboardy
- [x] **WIN-05**: Windows agent simulates Ctrl+V keystroke using robotjs
- [x] **WIN-06**: Windows agent adds 50-100ms delay between clipboard write and paste
- [x] **WIN-07**: Windows agent verifies clipboard content before pasting
- [x] **WIN-08**: Windows agent retries paste if verification fails

### Desktop Agent (Linux X11)

- [x] **LIN-01**: Linux agent connects to backend via WebSocket
- [x] **LIN-02**: Linux agent registers with hostname-based deviceId
- [x] **LIN-03**: Linux agent receives text via WebSocket messages
- [x] **LIN-04**: Linux agent writes text to clipboard using clipboardy
- [x] **LIN-05**: Linux agent simulates Ctrl+V keystroke using xdotool
- [x] **LIN-06**: Linux agent detects X11 display server at runtime
- [x] **LIN-07**: Linux agent uses X11-compatible clipboard and keyboard tools

### Connection Resilience

- [x] **RES-01**: Mobile app queues transcriptions locally when offline
- [x] **RES-02**: Mobile app replays queued transcriptions on reconnect
- [x] **RES-03**: Mobile app implements exponential backoff reconnection (1s -> 30s max)
- [x] **RES-04**: Desktop agent implements exponential backoff reconnection (1s -> 30s max)
- [x] **RES-05**: Desktop agent responds to heartbeat ping with pong
- [x] **RES-06**: Desktop agent detects missed pongs and reconnects
- [x] **RES-07**: Backend implements message acknowledgment protocol
- [x] **RES-08**: Mobile app marks transcriptions as sent only after ACK received

### Text Delivery

- [x] **DEL-01**: End-to-end latency from tap-stop to paste under 2 seconds (P95)
- [x] **DEL-02**: Desktop agent auto-pastes at current cursor position
- [x] **DEL-03**: Mobile app provides silent success (no confirmation dialogs)
- [x] **DEL-04**: Desktop agent falls back to clipboard-only if paste fails
- [x] **DEL-05**: Desktop agent logs paste events for debugging
