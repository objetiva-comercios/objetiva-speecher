## Design System

| Property | Value |
|----------|-------|

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-icon gaps within nav bar |
| sm | 8px | Internal padding within nav items |
| md | 16px | Nav bar horizontal padding, content area padding |
| lg | 24px | Nav bar height padding (top internal space) |
| xl | 32px | Not used this phase |
| 2xl | 48px | Nav bar total visible height (excluding safe area) |
| 3xl | 64px | Total nav bar allocation with safe area on most devices |

Exceptions:
- Nav bar visible height: 48px (content area) + safe-area-inset-bottom (variable per device)
- Raised mic circle: 56px diameter (overlaps nav bar by 16px above top edge)
- Touch target minimum: 44px for all tappable nav items (accessibility)

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | gray-100 (#f3f4f6) | App background, tab content area background |
| Secondary (30%) | white (#ffffff) | Nav bar background, cards, input fields |
| Accent (10%) | blue-500 (#3b82f6) | Active nav icon, raised mic circle fill, focus rings |
| Destructive | red-500 (#ef4444) | Recording state only (existing RecordButton behavior) |

Accent reserved for:
- Active tab icon color (the currently selected tab)
- Raised mic circle background fill
- Focus ring on interactive elements
- Loading spinner border

Additional semantic colors:
- gray-400 (#9ca3af): Inactive tab icons, disabled text, placeholder text
- gray-200 (#e5e7eb): Nav bar top border separator
- orange-500 (#f97316): Text editing mode indicator (existing pattern from RecordButton)
- green-600 (#16a34a): Success/sent status indicators (existing pattern from HistoryList)

Source: CONTEXT.md locked decisions + existing codebase color usage.

### Recording Pulse Animation

| Property | Value |
|----------|-------|
| Trigger | Global recording state is active (isRecording === true) |
| Target | Center mic nav circle only |
| Animation | CSS keyframe: scale(1) to scale(1.08) with blue-400 box-shadow glow |
| Duration | 1.5s infinite ease-in-out |
| Implementation | CSS @keyframes in index.css, applied via conditional class |

```css
@keyframes nav-mic-pulse {
  0%, 100% {
    box-shadow: 0 0 0 0 rgba(59, 130, 246, 0.5);
    transform: scale(1);
  }
  50% {
    box-shadow: 0 0 0 8px rgba(59, 130, 246, 0);
    transform: scale(1.08);
  }
}
.animate-nav-mic-pulse {
  animation: nav-mic-pulse 1.5s ease-in-out infinite;
}
```

## Interaction States

### Tab Switching

| State | Behavior |
|-------|----------|
| Default on launch | Speech tab (center) is active |
| Tap inactive tab | Instantly swap visible content (no animation) |
| Tap active tab | No-op |
| Switch while recording | Auto-stop recording, trigger auto-send, then switch tab |

### Active Tab Indication

| State | Visual |
|-------|--------|
| Active (History/Config) | Icon color changes to blue-500 |
| Active (Center Mic) | Blue circle already prominent; add ring-2 ring-blue-300 |
| Inactive | Icon color gray-400 |

### Double-Tap on Center Mic

| State | Behavior |
|-------|----------|
| On Speech tab, idle | Enter text editing mode (set buttonMode to 'text', show TranscriptionEditor) |
| On other tab | Switch to Speech tab first, then enter text editing mode |
| While recording | Ignored (existing RecordButton handles recording state) |

## Accessibility Contract

| Requirement | Implementation |
|-------------|----------------|
| Tab bar role | `role="tablist"` on nav container |
| Tab items role | `role="tab"` with `aria-selected` |
| Tab panels role | `role="tabpanel"` on content areas |
| Active indication | `aria-selected="true"` on active tab |
| Touch targets | Minimum 44px x 44px on all tab items |
| Screen reader labels | aria-label on each tab (Spanish: Historial, Voz, Configuracion) |
| Reduced motion | Respect `prefers-reduced-motion` — disable pulse animation |
